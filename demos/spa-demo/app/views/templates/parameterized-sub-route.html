This view is is subroute from route /parameterized

Routes can have subroutes. In this example route /parameterized have this sub-soute /parameterized/sub-route whicch displays this view!

You can have as many sub-routes as you like. However, of lower level route is paramterized and it receives first parameter matching next sub-route name it will be interpreted as next sub-route.

In this example if route /parameterized receives first parameter with value sub-route it will be interpreted as route /parameterized/sub-route

View location: 

/app/views/templates/parameterized-sub-route.html

Route definition: 
"/parameterized/sub-route": {
    view: "template!views/templates/parameterized-sub-route.html", 
    paramsMap: (...params) => {
        if (params.length > 1) {
            return false;
        }
        return {
            firstAndOnly: params[0]
        };
    },
    data: {
        title: "Parameterized sub route",
        category: "templates"
    }
}

Route accepts one optional, named paramater:
name: firstAndOnly, type: string

Current value:
firstAndOnly: ${this.firstAndOnly}
